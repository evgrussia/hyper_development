# User Stories

## Persona: Заказчик MVP

**Goals:** Быстро заказать MVP с нужными модулями.  
**Pain points:** Непонятно, что входит в проект; хочется выбрать только нужные модули.

### US-1 Просмотр портфолио
**As a** заказчик MVP, **I want to** просматривать примеры работ на лендинге, **so that** оценить уровень и тематику проектов.

**Acceptance criteria:**
- Given лендинг открыт, When пользователь скроллит до секции «Портфолио», Then отображаются кейсы (карточки с названием, описанием, ссылкой/скриншотом).
- Given кейсов много, When экран узкий, Then карточки адаптируются (сетка/карусель).

### US-2 Выбор модулей и отправка заказа
**As a** заказчик MVP, **I want to** заполнить форму заказа и выбрать модули из списка, **so that** отправить заявку и получить ответ в Telegram.

**Acceptance criteria:**
- Given пользователь на лендинге, When нажимает CTA «Заказать» / переходит к форме, Then видит форму: контакт (email/телефон/имя), описание задачи, выбор модулей (чекбоксы/мультиселект из списка из админки).
- Given форма заполнена валидно, When пользователь отправляет форму, Then заказ сохраняется в БД, владельцу приходит уведомление в Telegram (webapp), пользователю показывается сообщение об успешной отправке.
- Given форма заполнена невалидно, When пользователь отправляет форму, Then показываются ошибки валидации, заказ не отправляется.

## Persona: Бизнес-заказчик

**Goals:** Понять спектр услуг и оставить заявку.

### US-3 Просмотр услуг
**As a** бизнес-заказчик, **I want to** видеть список услуг на лендинге, **so that** понять, подходит ли рефакторинг/интеграции/другое.

**Acceptance criteria:**
- Given лендинг открыт, When пользователь просматривает секцию «Услуги», Then отображаются услуги (название, краткое описание), данные берутся из админки.

### US-4 Отправка заказа без выбора модулей (опционально)
**As a** бизнес-заказчик, **I want to** отправить заявку с произвольным описанием, **so that** описать задачу не из списка модулей.

**Acceptance criteria:**
- Given форма заказа открыта, When модули не выбраны, Then можно отправить заказ с полем «Описание»; заказ сохраняется и уходит в Telegram.

## Persona: Владелец (админ)

**Goals:** Управлять контентом лендинга и обрабатывать заказы; получать заказы в Telegram.

### US-5 Управление портфолио
**As a** владелец, **I want to** добавлять, редактировать и удалять кейсы в админке, **so that** актуализировать портфолио на лендинге.

**Acceptance criteria:**
- Given админ авторизован, When переходит в раздел «Портфолио», Then видит список кейсов; может создать/редактировать/удалить кейс (название, описание, ссылка, изображение, порядок).
- Given кейс сохранён, When лендинг загружен, Then секция «Портфолио» отображает актуальные данные.

### US-6 Управление услугами
**As a** владелец, **I want to** управлять списком услуг в админке, **so that** менять описание услуг на лендинге без кода.

**Acceptance criteria:**
- Given админ авторизован, When переходит в «Услуги», Then может CRUD услуги (название, описание, порядок). Изменения отражаются на лендинге.

### US-7 Управление секциями лендинга
**As a** владелец, **I want to** управлять секциями лендинга (вкл/выкл, порядок, контент), **so that** менять структуру и тексты без деплоя.

**Acceptance criteria:**
- Given админ авторизован, When переходит в «Секции лендинга», Then может включать/выключать секции, менять порядок и при необходимости контент (зависит от реализации: блоки или конфиг).

### US-8 Управление модулями заказа
**As a** владелец, **I want to** редактировать список модулей в форме заказа, **so that** предлагать актуальный набор без изменения кода.

**Acceptance criteria:**
- Given админ авторизован, When переходит в «Модули заказа», Then может CRUD модули (название, описание, порядок). В форме заказа на лендинге отображается актуальный список.

### US-9 Управление заказами
**As a** владелец, **I want to** видеть список заказов и менять их статус в админке, **so that** вести учёт и не терять заявки.

**Acceptance criteria:**
- Given админ авторизован, When переходит в «Заказы», Then видит список заказов (дата, контакт, выбранные модули, описание, статус); может открыть детали и изменить статус (новый, в работе, выполнен, отменён).
- Given новый заказ создан с лендинга, When владелец открывает Telegram webapp, Then видит уведомление о новом заказе (или список заказов).

### US-10 Получение заказов в Telegram
**As a** владелец, **I want to** получать уведомления о новых заказах в Telegram (webapp), **so that** оперативно реагировать без обязательного захода в админку.

**Acceptance criteria:**
- Given заказ отправлен с лендинга, Then в Telegram (bot webapp) владельцу приходит уведомление с основными данными заказа (контакт, модули, описание) и ссылкой в админку при необходимости.

## Приоритизация (MoSCoW)

- **Must:** US-1, US-2, US-5, US-6, US-8, US-9, US-10; базовая реализация US-7 (секции из админки).
- **Should:** US-3, US-4 (уже покрыты формой), полное управление секциями, статусы заказов.
- **Could:** Предпросмотр лендинга, расширенные фильтры заказов в админке.
- **Won't (этот релиз):** Личный кабинет заказчика, мультиязычность, платежи.
