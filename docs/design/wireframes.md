### 5. Форма заказа (#order)

| Элемент | Описание | Состояния |
|--------|----------|-----------|
| Заголовок | «Оставить заявку» / «Заказать проект» | — |
| Поле «Имя» | Текст, обязательное | Default, Focus, Error, Filled |
| Поле «Контакт» | Email и/или телефон (одно или два поля), обязательное | Default, Focus, Error, Filled |
| Поле «Описание» | Textarea, опционально | Default, Focus, Filled |
| Блок «Модули» | Список чекбоксов или мультиселект (данные из админки) | Default, Checked, Error (если обязательный выбор — по решению) |
| Кнопка «Отправить» | Submit | Default, Hover, Focus, Loading, Disabled после отправки |
| Сообщение об успехе | «Заявка отправлена. Мы свяжемся с вами в Telegram.» | После успешной отправки |
| Сообщение об ошибке | Текст ошибки валидации или сети | При ошибке |

**Layout:** Форма в один столбец; на десктопе можно узкий блок по центру. Адаптив: на мобильных форма на всю ширину с отступами.

**Валидация:** Обязательные поля — имя, контакт (email или телефон); описание и модули — по решению (модули можно не делать обязательными).

## States Summary

- **Empty:** портфолио/услуги/модули/заказы — заглушка «Добавьте первую запись» / «Заказов пока нет».
- **Loading:** скелетоны или спиннер при загрузке списков и при отправке формы.
- **Error:** валидация формы (подсветка полей + текст); сетевая ошибка (сообщение + повторить).
- **Success:** форма заказа — «Заявка отправлена»; админка — «Сохранено» (тост или сообщение).
